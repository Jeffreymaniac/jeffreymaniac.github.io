Rules

  • I don't wanna make an inappropriate website like for adult content or shock sites. Ask somebody else to do that.
  • I'm not going to make a super big/ambitious project without something in return. By a big project I mean something like a social media website like Tiktok or a shopping website or something that's gonna stress me out. I wouldn't exactly be the guy to go to to make sites like that, although I'm capable, it's a lot of hard work which I'd need something in return for like a gift card or something idk. Things like that would also require you to provide me with the things nessecary for it, like databases to store social media posts and API keys for whatever extra functionality you'd like and whatnot. I can give pointers if you need help with stuff like that or picking a good option.
  • You will need to give me access to whatever you want to use to make your website so that I can make what you ask me to make. For example, if you want your-username.github.io, then make your own Github account and invite me to it as a collaborator. If you already have a URL, then give me stuff like the a name or whatever so that I can use either Glitch, Replit, or Github to develop it. Ultimately it is your responsibility to provide me with some stuff to make your site for you.
  • The bullet point above doesn't apply if you are just asking me for code that you'll put into your website yourself, like if you just need help with an issue in JavaScript and you need me to fix the error then I have no problem with that!
  • You have to credit me for my work on your site(s) and/or when using my code. Usually I'll put a notice at the bottom of the website that says "Made by Jeff/JaecadeJnight" with a link to my website. You may move it around and make it smaller but you may not delete it or style it so that it's invisible. You may also move it to a credits page or something like that, but it needs to be easy for people accessing your site to know that I made it.
